Naive Search

Time worstO(n·m)
Time bestO(n)
SpaceO(1)
PreprocessingNone
Slides pattern one position at a time. On mismatch, restarts from scratch. Worst case on repetitive patterns like "AAAB" in "AAAA…".

KMP (Knuth-Morris-Pratt)

TimeO(n+m)
SpaceO(m) — LPS array
PreprocessingBuild LPS in O(m)
Best forRepetitive patterns, streaming
Pre-processes pattern to build LPS array. On mismatch at j, jumps to lps[j-1] instead of restarting. Never moves text pointer backwards — ideal for streaming genomic data.

Rabin-Karp

Time avgO(n+m)
Time worstO(n·m)
SpaceO(1)
Best forMultiple pattern search
Rolling polynomial hash. Compares hash values first, verifies on match. Hash collisions cause worst case. Best when searching many patterns simultaneously.

Boyer-Moore

Time bestO(n/m)
Time avgO(n)
SpaceO(σ+m)
Best forLong patterns, large text
Scans pattern right-to-left. Bad-character heuristic: if char not in pattern, skip m positions. Often the fastest algorithm in practice for genomic search.

Z-Algorithm

TimeO(n+m)
SpaceO(n+m)
StructureZ-array of concat
StyleElegant, linear
Concatenates pattern+"$"+text, builds Z-array where Z[i] = longest match of prefix. Positions where Z[i]=m are matches. Clean and cache-friendly.

When to use which?

Small textNaive (no overhead)
Repetitive DNAKMP (no backtracking)
Multi-patternRabin-Karp
Long patternsBoyer-Moore
Clean codeZ-Algorithm
For genomic search (long text, medium pattern), Boyer-Moore usually wins. KMP for streams. Rabin-Karp for gene family searches.
